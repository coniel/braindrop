# TopicViewInstance

<Description>
  An interface for describing topic view instances.
</Description>

Topic view instances are view instances used to render topics. Each topic has at least one view instance associated with it. Topic view instances are resposible for storing data related to the layout and drop positioning within the view.

<InterfaceTable
  data={[
    {
      name: 'id',
      type: 'string',
      required: true,
      description: (
        <span>
          The ID of the <Code>TopicViewInstance</Code> document.
        </span>
      ),
    },
    {
      name: 'view',
      type: 'string',
      required: true,
      description:
        'The ID of the registered topic view rendered by this view instance.',
    },
    {
      name: 'topic',
      type: 'string',
      required: true,
      description: 'The ID of the topic to which this view belongs.',
    },
    {
      name: 'createdAt',
      type: 'Date',
      required: true,
      description:
        'The timestamp at which the view instance was created.',
    },
    {
      name: 'updatedAt',
      type: 'Date',
      required: true,
      description: (
        <span>
          The timestamp at which the view instance was last updated.
          Equal to the <Code>createdAt</Code> value if the view
          instance has not been updated.
        </span>
      ),
    },
  ]}
/>

Additionally, topic view instances support adding any custom data fields needed for the view.
