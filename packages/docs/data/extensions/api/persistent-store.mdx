# PersistentStore

<Description>
  An interface for retrieving and managing persistently stored data.
</Description>

The persistent store is a key-value store which is persisted between app restarts.

The persistent store has two scopes, `global` and `local`. The global scope is backed up and synced accross devices when using the Sync add-on. The local scope is app specific and is not synced across devices. By default, values are set/retrieved from the global scope.

Values are automatically namespaced by extension ID in order to prevent seperate extensions from overwiting eachother's data.

The persistent store is intended for small pieces of simple data. If your extension needs to store a large amount of data or individual documents, [register a resource](/docs/extensions/api/core#registerresource) insteasd.

## Hooks

Hooks provide access to persistently stored data from inside React components. Unlike data accessed using static methods, hooks will cause the component to rerender when the data changes.

### clear

Permanently deletes all data added by the extension from the persistent store. Dispatches a `persistent-store:update` event.

```ts
PersistentStore.clear(core: Core, store: 'local' | 'global' | 'both'): void
```

### clearCache

Clears all data (including data added by other extensions) from the global store cache. This does not delete the persisted data. Dispatches a `persistent-store:clear-cache` event.

Useful for forcefully refreshing the store from a storage adapter.

```ts
PersistentStore.clearCache(): void
```

`PersistentStoreData`

An object containing key-value pairs namespaced by extension ID.

Example

```ts
data = {
  global: {
    app: {
      topics: ['topic-1-id', 'topic-2-id'],
      tags: ['tag-id'],
    },
    extensionId: {
      foo: 'bar',
    },
  },
  local: {
    app: {
      view: {
        id: 'topic',
        title: 'Sailing',
        resource: {
          type: 'topic',
          id: 'topic-id',
        },
      },
    },
  },
};
```

`PersistentStoreChanges`

An object containing new field values namespaced by the extension ID which caused the update, supports [`FieldValue`](/docs/extensions/api/utils#fieldvalue) mutations.

Example

```ts
changes = {
  extensionId: {
    foo: 'bar',
  },
};
```
