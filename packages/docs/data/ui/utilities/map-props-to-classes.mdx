# mapPropsToClasses

<Description>Maps a component's props to class names.</Description>

<Highlights
  features={[
    'Supports boolean and string props',
    'Appends className prop to allow extending class names',
    'Optional root class applied before other classes',
  ]}
/>

## Usage

Import the function from `@minddrop/utils`.

```tsx
import { mapPropsToClasses } from '@minddrop/utils';

export interface ButtonProps {
  size?: 'small' | 'medium' | 'large';
  color?: 'primary' | 'danger';
  disabled?: boolean;
}

export const Button: React.FC<ButtonProps> = ({
  children,
  className,
  color,
  disabled,
  size = 'medium',
}) => (
  <button
    className={mapPropsToClasses(
      { className, size, color, disabled },
      'button',
    )}
  >
    {children}
  </button>
);
```

A `Button` component with default props would render as:

```jsx
<button class="button medium" />
```

A `Button` component with the following props:

```jsx
<Button
  disabled
  size="small"
  color="primary"
  className="custom-button"
/>
```

Would render as:

```jsx
<button class="button small primary disabled custom-button" />
```

## Class names

Boolean props are mapped simply as the prop name. For example, a `disabled` prop would add a `disabled` class when `true`.

String props are mapped as the prop's value. For example, a `size` prop with a value of `"small"` would add a class of `small`.

The `className` prop is appended to the generated class names.

Falsy props do not generate any classes.

The `rootClass` value is prepended to the generated class names.

`class="rootClass [prop classes] className"`

## API Reference

```ts
mapPropsToClasses(
  props: Record<string, string | boolean>,
  rootClass?: string,
);
```
